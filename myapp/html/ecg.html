Abstract

This Jupyter notebook contains my implementation of a Cardiac Arrhythmia diagnosis system using machine learning models trained on processed electrocardiogram signals. The 96 half-hour long signals were obtained from 47 patients at the BID Medical Center in Boston from 1975 to 1979; the signals were published as part of the MIT-BIH Arrhythmia Database in 2005. To create the diagnosis system, I first cleaned the data, reshaping it into a tabular format and discarding outlier instances. I then engineered features by extracting individual heartbeats from the signals, before performing Principal Component Analysis on the dataset. Finally, I performed cross-validation on several models, fine-tuned the hyperparameters of the best performers, and combined their performance into a single model through ensembling.

R-peaks are essentially the heartbeats' upper peaks; the signal values between one R-peak and the next will constitute one beat.

def extract_heartbeats(signal):
    rpeaks = ecg.ecg(signal, sampling_rate=360, show=False)["rpeaks"]
    return np.split(signal, rpeaks)[1:-1]

def adjust_labels(records, labels):
    new_data = []
    for i, record in enumerate(records):
        beats = extract_heartbeats(record[2])
        label = labels[i // 2][2]
        difference = len(label) - len(beats)

        if not difference:
            new_data.append(record + [label])
        elif 0 < difference < 100:
            new_data.append(record + [label[:-difference]])
    return new_data

2.4) DataFrame Construction

Next, I constructed a DataFrame using the dataset, where each instance (row) represents one heartbeat. Different beats have different durations, so I recorded each beat as exactly 1 second long - almost all beats were shorter. The ECG sampling rate is 360Hz, meaning the DataFrame contains "360" features - the signal values at each of the 360 frames. I discarded heartbeats exceeding 1 second in length, and padded those shorter than 1 second with zeros. Meanwhile, the label values are single characters: N represents a normal beat, while all other characters represent an anomaly. To simplify modeling, I used a binary classification task - to predict where the beat is normal. The final DataFrame consists of the 360 features, labels, patient IDs, and channel names: the latter 2 fields are for diagnosing patients and are excluded from modeling.

In[20]:
def engineer_heartbeats(signal, labels):
    beats = extract_heartbeats(signal)
    new_data, new_labels = [], []
    for i, beat in enumerate(beats):
        if len(beat) <= 360:
            new_data.append(np.pad(beat, (0, 360 - len(beat))))
            new_labels.append(labels[i])
    return pd.DataFrame(new_data), pd.Series(new_labels)
In[21]:
def construct_dataframe(data):
    colnames = ["patient", "channel", "abnormal"] + list(range(360))
    new_data = pd.DataFrame(columns=colnames)
    for patient, channel, signal, labels in data:
        df, new_labels = engineer_heartbeats(signal, labels)
        df.insert(0, "patient", patient)
        df.insert(1, "channel", channel)
        df.insert(2, "abnormal", (new_labels != "N").astype(int))
        new_data = pd.concat([new_data, df])
    return new_data

2.5) Train-Test Splitting

Having manipulated the data into a traditional format, I split the dataset into a training set and an evaluation set. I wanted both sets to have equal proportions of regular vs irregular heartbeats, which I achieved by stratifying the split. I allocated 80% of the dataset for training, leaving 20% for evaluation.

2.6) Principal Component Analysis

Most heartbeats in the dataset are shorter than 1 full second, making the last few columns extremely sparse, with explained variance not worth their additional computation time. Thus, as the last preprocessing step, I performed Principal Component Analysis on the dataset, retaining 95% of its explained variance but dramatically reducing its number of features.

3. Modeling

In this section, I trained several models, tuned the hyperparameters of the 3 best performers, and ensembled the 2 best performers into a single model.

3.1) Initial Models

Using 5-fold cross-validation, I first trained and evaluated several models, listed below, with default parameters. The 3 best performers, i.e. those with the highest mean F1-score, were Random Forest, Support Vector (RBF kernel), and Gradient Boost classification. In the context of medical diagnosis, low recall means many undiagnosed patients, while low precision means many falsely diagnosed patients. As these consequences are both extremely severe, I sought to minimize both simultaneously, so I chose F1 as the scoring metric.
