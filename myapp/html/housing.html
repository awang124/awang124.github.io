California House Price Prediction¶

  • Author: Andy Wang, Johns Hopkins University, Class of 2027
  • Github Repository: https://github.com/sirandy3000/projects/tree/master/housing

Abstract¶

This Jupyter notebook contains my solution to a classic introductory machine learning problem: to predict the median house price of 20640 California districts. The features (median income, population, location, etc.) were obtained as part of the 1990 US census. To solve this problem, I first performed Exploratory Data Analysis, considering each feature's distribution and correlation with median house price, to better understand the possible insights that each feature could provide. I then engineered and transformed features using techniques such as K-Means clustering and Nearest-Neighbors imputation. Finally, I performed cross-validation on several models and fine-tuned the hyperparameters of the best performer.

3.4) Feature Engineering¶

I wanted to create a measure of "urban-ness" using latitude and longitude, and initially thought the best way to achieve this was to measure the distances between each district and its nearest neighbors. I used KNeighborsClassifier().kneighbors() to calculate these distances, but found that urban and rural districts yielded similar results.

I then realized I could measure the distances between each district and the urban centers (i.e. San Francisco and Los Angeles) as "urban-ness". I could cluster the districts using KMeans, using each cluster center as an urban center, and the distances and new features. I chose RBF similarity as the distance metric, as Euclidean distance again yielded similar results for all districts.

In [16]:
class UrbanDistance(BaseEstimator, TransformerMixin):
    def __init__(self, k=5, gamma=1, random_state=42):
        self.k = k
        self.gamma = gamma
        self.random_state = random_state
    def fit(self, X, y=None):
        self.kmeans = KMeans(self.k, random_state=self.random_state)
        self.kmeans.fit(X)
        return self
    def transform(self, X):
        return rbf_kernel(X, self.kmeans.cluster_centers_, gamma=self.gamma)
    def get_feature_names_out(self, feature_names_in=None):
        return [f"center_{i}_distance" for i in range(self.k)]

3.5) Final Steps¶

The final preprocessing steps are to impute missing values, encode categorical features, and scale quantitative features. Feature total_bedrooms contains missing values, for which I used nearest-neighbor imputation, as districts with similar total_rooms, population, and households likely have similar total_bedrooms. The values of ocean_proximity, listed in section (2.2), have no innate ordering, so I used one-hot encoding. Finally, I assembled all steps into a single transformer.

5.2) Feature Importances¶

The most important features in predicting median_house_value are median income, whether the district is INLAND, and the district's distances to the urban centers. Median income is sensible, as wealthier people tend to buy more expensive houses; "inland-ness" is surprising, however, and is worth further investigation in future analyses.
